Apa itu REST API?

REST (Representational State Transfer) adalah arsitektur komunikasi antar sistem menggunakan protokol HTTP. REST API memungkinkan client (seperti browser atau aplikasi mobile) berinteraksi dengan server melalui operasi CRUD (Create, Read, Update, Delete) yang dipetakan ke method HTTP.

Setiap resource (data) diakses melalui URL yang unik, dan server mengembalikan data dalam format seperti JSON. Playground ini mensimulasikan server REST API di dalam browser — semua data disimpan di memori lokal.

HTTP Methods

Method HTTP menentukan jenis operasi yang ingin dilakukan terhadap resource:

GET Baca Data

Mengambil data dari server. Tidak mengubah data apapun. Bisa menerima query parameter untuk filtering.

POST Buat Baru

Membuat resource baru. Body request berisi data baru yang akan disimpan. Server mengembalikan data yang sudah dibuat beserta ID-nya.

PUT Replace Semua

Mengganti seluruh data resource yang sudah ada. Semua field harus dikirim — yang tidak dikirim akan dihapus.

PATCH Update Sebagian

Mengubah sebagian field dari resource. Hanya field yang dikirim yang akan diupdate, sisanya tetap.

DELETE Hapus

Menghapus resource dari server. Biasanya mengembalikan status 204 (No Content) atau data yang dihapus.

HEAD Metadata Saja

Sama seperti GET tapi hanya mengembalikan header, tanpa body. Digunakan untuk mengecek resource tanpa mendownload datanya.

Status Code

Status code adalah angka 3 digit yang menjelaskan hasil dari request:

2xx Sukses

200 OK — Request berhasil
201 Created — Resource baru berhasil dibuat
204 No Content — Berhasil tapi tidak ada data dikembalikan

4xx Client Error

400 Bad Request — Data tidak valid
404 Not Found — Resource tidak ada
422 Unprocessable — Data valid tapi tidak bisa diproses

5xx Server Error

500 Internal Server Error — Kesalahan di sisi server
503 Service Unavailable — Server sedang tidak bisa melayani

Tips Belajar

Gunakan Playground di tab pertama untuk mencoba langsung setiap konsep:

1. Klik endpoint di sidebar → otomatis isi method & URL 2. Klik "Kirim" → lihat response (status code, body, waktu) 3. Coba POST untuk buat data baru, lalu GET untuk lihat hasilnya 4. Coba PUT/PATCH untuk update, DELETE untuk hapus 5. Coba kirim data yang salah → perhatikan error 400/404 6. Periksa tab Riwayat untuk melihat semua request yang sudah dilakukan
